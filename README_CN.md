# 8D报告处理工具

> 一款用于8D问题解决报告审核与翻译的TRAE WORK技能，专为制造业质量工程人员打造。

[English Documentation](./README.md)

---

## 功能简介

本技能以两种模式处理8D（八步法）问题解决报告：

| 模式 | 功能 | 输出 |
|------|------|------|
| **审核模式** | 按D1-D8行业标准逐项校验8D报告，识别信息缺失、逻辑漏洞、术语错误 | Word文档，包含问题清单和修改建议 |
| **翻译模式** | 使用行业专属术语库，将定稿8D报告在中英文之间互译 | 保留原文格式、数据和排版的翻译文档 |

技能根据用户输入的关键词自动识别意图并切换模式。意图不明确时，默认进入审核模式，审核完成后再询问是否需要翻译。

## 覆盖行业

内置四大制造行业的术语对照库和审核重点清单：

1. **汽车供应链** - 注塑、冲压、线束、气密泄漏、尺寸超差、PFMEA/MSA/PPAP、工装、治具、异响、喷涂不良
2. **消费电子**（手机/平板/笔记本/穿戴设备） - 主板、FPC、摄像头模组、点胶、虚焊、ESD静电、跌落测试、卡合间隙、电池鼓包
3. **通用电子信息/PCB** - PCB短路/开路、锡珠、元器件偏移、阻抗、EMC、连接器、耐压测试
4. **LCD/OLED显示面板** - 亮点、暗点、Mura云斑、漏光、残影、色度偏移、FOG/COG绑定、偏光片、背光模组、切割崩边、液晶气泡

## 核心功能

### 审核模式
- 严格按D1-D8标准逐项校验所有章节
- 自动识别三类问题：信息缺失、逻辑漏洞、文本/术语问题
- 中文报告额外检查：模糊口语化、术语前后不一致、缺乏量化数据
- 英文报告额外检查：语法时态、术语拼写、长句通顺度、章节标题规范性
- 标注跨行业术语误用（如汽车报告中出现面板术语）
- 生成结构化Word报告，包含合规评级、问题汇总表、逐条修改建议、高频踩坑提醒

### 翻译模式
- D1-D8章节使用固定标准英文标题，全程统一
- 自动识别文档所属行业，匹配对应术语库，前后翻译一致
- 完整保留所有数字、批次号、料号、不良数量、时间节点、表格、图片和排版
- 输出符合外企制造业书面语态的专业译文，适合海外客户SQE审阅

### 模式切换说明

技能根据消息关键词自动判断模式：

| 关键词 | 触发模式 |
|--------|----------|
| 审核、校验、检查、审查、问题、整改、合规、review、audit、check | 审核模式 |
| 翻译、中英互译、译文、英文版、中文版、translate | 翻译模式 |

如果同时提及审核和翻译，或意图不明确，优先进入审核模式。审核完成后会询问是否需要翻译。

## 输出说明

### 审核模式输出
Word文档，包含以下内容：
1. **报告基础信息** - 识别的行业、语言类型、整体合规评级（合格 / 部分整改 / 严重缺失）
2. **问题汇总总表** - 序号、所属D模块、问题类型（缺失/逻辑/语法术语）、问题原文说明
3. **分模块逐条修改建议** - 针对每条问题给出具体补充和改写方案，同步提供行业标准术语
4. **高频踩坑总结** - 各行业客户审核常驳回的同类问题提醒

### 翻译模式输出
与原文件格式相同的翻译后8D文档（PDF/Word/PowerPoint），保留所有图片、表格和排版。

## 技术信息

- **技能名称**：`8d-report-processor`
- **MCP依赖**：File System（用于读取PDF/Word/TXT/MD/PPT附件）
- **支持输入格式**：纯文本、PDF、Word（.docx）、PowerPoint（.pptx）、TXT、Markdown
- **支持语言**：简体中文、英文
- **D1-D8标准**：基于汽车行业8D方法论，适配四大制造行业

## 使用限制

- 审核模式不执行翻译操作，仅输出问题与整改建议
- 翻译模式不执行审核操作，假定源文档已定稿
- 技能不编造8D内容，仅基于用户提供的原文进行处理
- 若源文档存在明显逻辑缺失或数据空白，翻译模式会提示建议先完成审核整改
